## Slide 8｜資料庫結構（一）
### Database Structure — Entity Relationship

```
users                    construction_records
─────────────            ────────────────────
user_id   PK             record_id   PK  ←──────────────┐
username  UNIQUE         record_date UNIQUE              │
password                 workers_count                   │
role                                                     │
                                              record_materials
                         materials            ───────────────────
                         ─────────────        id          PK
                         material_id PK ──→   record_id   FK → construction_records
                         material_name        material_id FK → materials
                         unit                 quantity
                                              UNIQUE(record_id, material_id)
```

**關係說明 Relationships**
- `construction_records` ↔ `materials`：多對多 (Many-to-Many)，透過 `record_materials` 關聯表實現
- 刪除 `construction_records` 一筆 → `record_materials` 對應列 CASCADE 自動刪除
- 刪除 `materials` 一種 → `record_materials` 對應列 CASCADE 自動刪除

## Slide 10｜資料庫結構（三）
### Database Design Decisions — 設計決策說明

**為何使用關聯表取代硬編碼欄位？**
Why use a junction table instead of fixed columns?

- v1 做法：`construction_records` 直接含 `cement_used`、`steel_used` 欄位
  → 問題：新增材料種類需修改 schema，擴充性差
- v2 做法：獨立 `materials` 目錄 + `record_materials` 多對多關聯
  → 優點：材料種類可動態新增，不需更動資料表結構

**Pivot 查詢策略**
- `get_all_records_with_materials()` 使用 `pandas.pivot_table()` 將長表轉寬表
- 材料欄位動態展開，新增材料後自動出現於報表與匯出 CSV

**ID 重排機制設計**
Two-Pass Negative ID Reordering Algorithm
- 直接更新主鍵會觸發 UNIQUE 衝突
- 解法：第一輪改為負數暫存 ID → 第二輪改回正數最終 ID
- 同步更新 `sqlite_sequence` 確保 AUTOINCREMENT 從正確值繼續

# ① 資源快取：整個 App 生命週期只建立一個 ProjectManager
@st.cache_resource
def get_manager() -> ProjectManager:
    return ProjectManager()

# ② 登入驗證：核對帳密後寫入 session_state
def render_login():
    with st.form("login_form"):
        username = st.text_input("帳號")
        password = st.text_input("密碼", type="password")
        submitted = st.form_submit_button("登入", type="primary")

    if submitted:
        user_info = get_manager().authenticate(username.strip(), password)
        if user_info:
            st.session_state["logged_in"] = True
            st.session_state["user"] = user_info
            st.rerun()                        # ← 切換至主畫面
        else:
            st.error("帳號或密碼錯誤")

# ③ 主入口：依角色分流
def main():
    if not st.session_state.get("logged_in"):
        render_login()
    elif st.session_state["user"]["role"] == "admin":
        render_admin_interface()
    else:
        render_user_interface()

- `@st.cache_resource`：Singleton 模式，防止每次重新渲染時重複建立資料庫連線

## Slide 12｜原型程式碼（二）
### Prototype Code — 新增施工紀錄

```python
def add_record(self, record: ConstructionRecord,
               material_quantities: dict) -> tuple[bool, str]:
    try:
        with self._get_connection() as conn:
            cursor = conn.cursor()

            # ① 寫入施工日誌主表
            cursor.execute(
                "INSERT INTO construction_records (record_date, workers_count)"
                " VALUES (?, ?)",
                (record.date, record.workers),
            )
            new_id = cursor.lastrowid       # ← 取得剛新增的主鍵

            # ② 逐一寫入各材料消耗量（quantity > 0 才寫入）
            for mat_id, qty in material_quantities.items():
                if qty > 0:
                    cursor.execute(
                        "INSERT OR IGNORE INTO record_materials"
                        " (record_id, material_id, quantity) VALUES (?, ?, ?)",
                        (new_id, int(mat_id), float(qty)),
                    )
            conn.commit()
            return True, f"✅ {record.date} 的施工日誌新增成功！"

    except sqlite3.IntegrityError:          # ← 捕捉日期重複例外
        return False, f"⚠️ {record.date} 的施工紀錄已存在"
```

**重點說明**
- 參數化查詢（`?` 佔位符）：防止 SQL Injection
- `lastrowid`：取得資料庫自動產生的主鍵，用於關聯表寫入
- `IntegrityError`：`record_date UNIQUE` 約束觸發時捕捉，回傳友善訊息而非程式崩潰
- 原子性：主表與子表在同一 `conn.commit()` 中提交，確保資料一致

## Slide 13｜原型程式碼（三）
### Prototype Code — ID 重排演算法

```python
def _reorder_record_ids(self) -> None:
    with self._get_connection() as conn:
        cursor = conn.cursor()
        cursor.execute("PRAGMA foreign_keys = OFF")   # ← 暫時關閉外鍵約束

        rows = cursor.execute(
            "SELECT record_id FROM construction_records ORDER BY record_date ASC"
        ).fetchall()

        if not rows:
            return

        # ① 第一輪：舊 ID → 負數暫存 ID（避免 UNIQUE 衝突）
        for new_id, row in enumerate(rows, start=1):
            old_id = row["record_id"]
            if old_id != new_id:
                cursor.execute("UPDATE record_materials SET record_id=? WHERE record_id=?",
                               (-new_id, old_id))
                cursor.execute("UPDATE construction_records SET record_id=? WHERE record_id=?",
                               (-new_id, old_id))

        # ② 第二輪：負數暫存 ID → 正數最終 ID
        for new_id in range(1, len(rows) + 1):
            cursor.execute("UPDATE record_materials SET record_id=? WHERE record_id=?",
                           (new_id, -new_id))
            cursor.execute("UPDATE construction_records SET record_id=? WHERE record_id=?",
                           (new_id, -new_id))

        # ③ 重設 AUTOINCREMENT 計數器
        cursor.execute("UPDATE sqlite_sequence SET seq=? WHERE name='construction_records'",
                       (len(rows),))
        cursor.execute("PRAGMA foreign_keys = ON")
        conn.commit()
```

**演算法示意（Example）**

```
刪除前 Before：  ID = 1, 3, 5, 7
刪除 ID=3 後：   ID = 1, 5, 7

第一輪 Pass 1：  1(不動), 5→-2, 7→-3
第二輪 Pass 2：  -2→2, -3→3

重排後 After：   ID = 1, 2, 3  ✓
```

## Slide 14｜原型程式碼（四）
### Prototype Code — Pivot 查詢與動態報表

```python
def get_all_records_with_materials(self) -> pd.DataFrame:
    with self._get_connection() as conn:
        # ① 讀取施工紀錄主表
        records_df = pd.read_sql_query(
            "SELECT record_id, record_date, workers_count"
            " FROM construction_records ORDER BY record_date ASC", conn
        )
        # ② 讀取所有材料消耗（長表格式 Long Format）
        mat_df = pd.read_sql_query(
            """SELECT rm.record_id, m.material_name, rm.quantity
               FROM record_materials rm
               JOIN materials m ON rm.material_id = m.material_id""", conn
        )

    # ③ Pivot：長表 → 寬表（每種材料獨立一欄）
    pivot = mat_df.pivot_table(
        index="record_id", columns="material_name",
        values="quantity", aggfunc="sum", fill_value=0
    )
    pivot.columns.name = None
    return records_df.merge(pivot.reset_index(), on="record_id", how="left").fillna(0)
```

**Long → Wide 轉換示意**

```
長表 (Long Format)          寬表 (Wide Format / Pivot)
─────────────────────       ─────────────────────────────────
record_id  material  qty    record_id  水泥  鋼筋  砂石
1          水泥       5      1          5     3     0
1          鋼筋       3  →   2          0     0     8
2          砂石       8      3          2     1     0
3          水泥       2
3          鋼筋       1
```
